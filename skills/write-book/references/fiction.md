# 小说写作协议

## 目录

- 故事事实层
- 全书与章节结构
- 场景设计
- 章节提交
- 连续性检查

## 故事事实层

把以下内容写入 `canon.md`，不要只留在聊天或正文里：

- 人物身份、欲望、恐惧、能力边界、关系和不可逆经历；
- 世界规则、地理、组织、物件和专有名词；
- 精确时间线、年龄、距离和因果约束；
- 叙述视角能够知道与不能知道的内容。

把尚未决定的内容标为待决，不要用多个互相冲突的版本占据事实层。需要改设定时先列出受影响章节，再更新事实层和正文。

## 全书与章节结构

用“欲望—阻力—选择—代价—新状态”描述主要叙事弧。每一章至少改变一个重要状态：人物理解、关系、资源、风险、目标或读者掌握的信息。如果删除一章不会改变后续状态，合并或重构该章。

控制信息释放：区分角色知道、视角人物知道、读者知道和作者知道。反转必须能够从早期文本回溯到公平线索，不能依赖事后新增规则。

## 场景设计

每个场景明确：

1. 谁想得到什么；
2. 当前阻力是什么；
3. 角色做出什么选择；
4. 选择付出什么代价；
5. 场景结束后什么发生了不可忽略的变化。

从最晚的有效进入点开始，在变化已经发生后尽快离场。用可观察行动、感官选择和具体语言承载情绪，不要把已经演出的内容再解释一遍。

## 章节提交

章节摘要必须记录：

- 场景顺序与主要因果；
- 人物状态和关系变化；
- 新增事实、物件、伤势、资源和时间推进；
- 新建、强化或兑现的伏笔；
- 下一章必须承接的压力或问题。

漂亮句子、完整对白和氛围描写不进入摘要，除非它们本身是必须保持一致的线索。

## 连续性检查

每 3–5 章核对一次：时间、地点、身体状态、物件归属、角色知识、关系状态、世界规则、叙述视角和未兑现伏笔。分部结束时检查人物弧是否由选择推动，而不是由作者说明推动。
