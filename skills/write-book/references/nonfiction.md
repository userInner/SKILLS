# 非虚构写作协议

## 目录

- 读者与核心承诺
- 主张—证据系统
- 章节论证
- 引用与不确定性
- 教材和方法书

## 读者与核心承诺

明确读者已经知道什么、相信什么、能够做什么，以及读完后应新增的理解或能力。全书只保留一个核心承诺；各分部和章节分别承担可验证的子承诺。

## 主张—证据系统

在 `research.md` 为可核验主张建立唯一 ID，记录证据、来源 ID、状态和使用章节。优先使用原始资料、权威机构和可追溯数据。二手材料可用于发现线索，但不要在未核验时把它升级为事实。

把内容分为：

- 已核验事实；
- 有来源支持但存在争议的解释；
- 作者的分析或推断；
- 例示性假设；
- 尚待核验的占位内容。

在正文中清楚区分这些类型。绝不制造引文、页码、统计数字、实验结果、人物经历或来源。

## 章节论证

每章采用可审计链条：读者问题 → 本章主张 → 证据/例子 → 可能反例或限制 → 推论 → 对下一章的需要。删除不能推进链条的资料堆积。

章节计划中列出“必须有证据支持的句子”和所需来源文件。生成上下文包时用 `--include` 加载相关资料，而不是把整个研究库塞入上下文。

## 引用与不确定性

保留来源标题、作者/机构、日期、URL 或本地文件位置以及访问日期（需要时）。直接引文必须回到原文核对；无法定位的引文改为转述或删除。对时效性事实重新联网核验，并标注核验日期。

来源相互冲突时不要强行合并，记录冲突、各自证据和采用某一解释的理由。缺少证据时缩小主张强度，而不是用流畅措辞掩盖不确定性。

## 教材和方法书

为每章定义学习目标、先修知识、示例、练习、常见误区和掌握标准。概念首次出现时定义，后续使用固定术语。示例必须覆盖正常路径和至少一个边界情况；步骤型内容必须由前置条件、动作、检查点和失败处理组成。
